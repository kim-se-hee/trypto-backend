# 송금

## 범위

- 국내 거래소(업비트, 빗썸)와 해외 거래소(바이낸스) 간 양방향 코인 송금이 가능하다
- KRW(법정화폐)는 송금할 수 없다. USDT는 암호화폐이므로 송금 가능하다

## 송금 절차

1. 도착 거래소에서 입금 주소를 조회한다 (코인, 체인 선택)
2. 출발 거래소에서 출금을 요청한다 (도착 주소 직접 입력, 체인 선택, 수량 입력)

## 체인

- 송금 시 체인을 반드시 선택해야 한다
- 양쪽 거래소 모두 해당 코인+체인 조합을 지원해야 송금이 가능하다
- 지원하지 않는 체인으로 전송하면 자금을 잃는다 (시뮬레이션에서는 동결로 표현)

## 출금 수수료

- 출금 수수료는 거래소, 코인, 체인별로 다르다
- 수수료는 송금 수량과 별도로 출발 지갑에서 차감된다 (수신측은 전액 수령)
- 출발 지갑의 가용 잔고가 송금 수량 + 수수료 이상이어야 한다

## 최소 출금 수량

- 거래소, 코인, 체인별로 최소 출금 수량이 존재한다
- 최소 출금 수량 미만의 송금 요청은 거부된다

## 입금 주소

- 송금 시 도착 거래소의 입금 주소를 직접 입력한다
- 입금 주소는 (지갑, 체인) 단위로 고유하다. 같은 체인의 모든 토큰은 같은 주소를 사용한다
- 태그(메모)가 필요한 코인이 있다 (XRP, XLM, EOS, ATOM 등). 태그를 누락하면 자금을 잃는다

## 멱등성

- 클라이언트가 `clientTransferId`(UUID)를 생성하여 전송한다
- 서버는 동일한 `clientTransferId`로 중복 요청이 들어오면 기존 송금 결과를 반환하고 새 송금을 생성하지 않는다

## 잔고 변동

- 성공 시: 출발 지갑에서 `deductAvailable(amount + fee)`, 도착 지갑에서 `addAvailable(amount)`
- 동결 시: 출발 지갑에서 `lock(amount + fee)`, 24시간 후 `unlock(amount + fee)`로 가용 잔고 복원

## 동결 (시뮬레이션)

잘못된 송금은 에러가 아니라 시뮬레이션 결과로 처리된다:

- **잘못된 주소**: 입력한 주소가 라운드 내 어떤 지갑의 입금 주소와도 일치하지 않으면 24시간 동결
- **잘못된 체인**: 도착 거래소가 해당 코인+체인을 지원하지 않으면 24시간 동결
- **태그 누락**: 태그가 필수인 코인에서 태그를 입력하지 않으면 24시간 동결

동결된 자금은 매매에 사용할 수 없다. 24시간 후 수수료를 포함하여 출발 지갑으로 자동 반환된다.
