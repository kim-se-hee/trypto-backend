# 복기 분석 공통 비즈니스 규칙

복기 리포트([regret-report.md](regret-report.md))와 복기 그래프([regret-chart.md](regret-chart.md))가 공통으로 참조하는 계산 규칙을 정의한다.

---

# 공통 용어

| 용어 | 의미 |
|------|------|
| `expected_amount` | 원칙을 지켰다면 거래했을 금액. "안 샀어야 했다"면 0, "이 시점에 팔았어야 했다"면 그때의 매도 금액 |
| `actual_amount` | 실제로 거래한 금액. 아직 보유 중이면 현재 평가 금액 |
| `loss_amount` | 원칙 위반이 만든 손익 차이. **양수면 손해**, 음수면 위반했지만 결과적으로 이득 |
| `기준가` | 위반 거래의 손익을 매길 때 비교 기준이 되는 가격. 결정 방법은 [기준가 결정 로직](#기준가-결정-로직) 참조 |

# 원칙별 손실 금액 계산

위반 유형에 따라 "원칙을 지켰다면" 시나리오의 금액을 산출한다. 모든 금액은 해당 거래소의 기축통화 단위(국내: KRW, 바이낸스: USDT)로 계산한다.

## 추격 매수 금지 (`CHASE_BUY_BAN`)

- **원칙 준수 시**: 해당 매수 주문을 하지 않았을 것
- **expected_amount**: 0 — 안 샀어야 하므로 투입 금액 없음
- **loss_amount**: 안 샀어야 할 코인을 산 뒤, 가격이 얼마나 빠졌는지(또는 올랐는지)

```
loss_amount = (체결가 - 기준가) × 체결수량

예) 1000원에 매수 → 이후 800원에 매도(기준가)
    loss_amount = (1000 - 800) × 수량 = 양수 → 손해

예) 1000원에 매수 → 이후 1200원에 매도(기준가)
    loss_amount = (1000 - 1200) × 수량 = 음수 → 위반했지만 이득
```

## 물타기 제한 (`AVERAGING_DOWN_LIMIT`)

- **원칙 준수 시**: 제한 횟수를 초과한 추가 매수를 하지 않았을 것
- **expected_amount**: 0 — 초과분은 안 샀어야 함
- **loss_amount**: 추격 매수 금지와 동일한 방식

```
loss_amount = (체결가 - 기준가) × 체결수량
```

## 과매매 제한 (`OVERTRADING_LIMIT`)

- **원칙 준수 시**: 제한 횟수를 초과한 주문을 하지 않았을 것
- **expected_amount**: 0 — 초과분은 거래하지 않았어야 함
- **loss_amount**: 초과 주문이 만든 손익. 매수와 매도의 계산 방향이 반대

```
매수 주문: loss_amount = (체결가 - 기준가) × 체결수량
매도 주문: loss_amount = (기준가 - 체결가) × 체결수량
```

- **매수 위반**: "안 샀어야 할 걸 샀다" → 위반분 우선 매칭으로 기준가를 결정한다 ([기준가 결정 로직](#기준가-결정-로직) 참조)
- **매도 위반**: "안 팔았어야 할 걸 팔았다" → 이미 판 거라 매칭할 게 없다. 기준가는 현재가를 사용한다

## 손절 미이행 (`LOSS_CUT`)

- **원칙 준수 시**: 손실률이 설정값에 도달했을 때 즉시 매도했을 것
- **expected_amount**: 손절 시점에 팔았다면 받았을 금액 = `평균 매수가 × (1 - 손절률/100) × 보유 수량`
- **actual_amount**: 실제 매도 금액. 아직 보유 중이면 현재 평가 금액

```
loss_amount = actual_amount - expected_amount

예) 손절 시점 매도가(expected) 90만원, 실제 매도가(actual) 70만원
    loss_amount = 70만 - 90만 = -20만 → 손절했으면 20만원 덜 잃었다
```

## 익절 미이행 (`PROFIT_TAKE`)

- **원칙 준수 시**: 수익률이 설정값에 도달했을 때 즉시 매도했을 것
- **expected_amount**: 익절 시점에 팔았다면 받았을 금액 = `평균 매수가 × (1 + 익절률/100) × 보유 수량`
- **actual_amount**: 실제 매도 금액. 아직 보유 중이면 현재 평가 금액

```
loss_amount = expected_amount - actual_amount

예) 익절 시점 매도가(expected) 130만원, 실제 매도가(actual) 110만원
    loss_amount = 130만 - 110만 = 20만 → 익절했으면 20만원 더 벌었다
```

# 기준가 결정 로직

`CHASE_BUY_BAN`, `AVERAGING_DOWN_LIMIT`, `OVERTRADING_LIMIT` (매수)의 수식에서 쓰이는 기준가를 정하는 방법.

## 위반분 우선 매칭

코인은 구분이 없으므로 어떤 코인이 매도되었는지 알 수 없다. 따라서 **위반 매수분이 먼저 매도된 것으로 간주한다.**

이 가정을 사용하는 이유:
- 코인은 하나하나 구분이 안 되므로, 어떤 순서로 매칭하든 "이게 진짜다"라고 할 근거가 없다. 그러니 우리 목적에 맞는 가정을 고르면 된다
- 위반분을 우선 매칭하면 매도가 발생할 때 위반의 영향이 바로 확정되어, "위반이 실제로 어떤 영향을 줬는가"를 빠르게 보여줄 수 있다

## 기준가 결정 절차

위반 매수 이후에 해당 코인의 매도 이력을 시간순으로 가져와서, 위반 매수 수량이 찰 때까지 매칭한다.

**1. 위반 수량이 전부 매칭된 경우 (전량 실현)**

매칭된 매도 체결가가 기준가가 된다. 값이 고정되므로 이후에는 변하지 않는다.

```
위반 매수: 20개 × 1000원
이후 매도: 50개 × 900원 → 위반 20개 전부 커버

loss_amount = (1000 - 900) × 20 = 2,000원 손해 (확정)
```

**2. 위반 수량이 일부만 매칭된 경우 (부분 실현)**

매칭된 부분은 매도 체결가로 확정, 남은 부분은 조회 시점의 현재가로 계산한다.

```
위반 매수: 20개 × 1000원
이후 매도: 10개 × 900원 → 위반 20개 중 10개 커버

실현분: (1000 - 900) × 10 = 1,000원 손해 (확정)
미실현분: (1000 - 현재가) × 10 (변동)
loss_amount = 실현분 + 미실현분
```

**3. 매도 이력이 없는 경우 (전량 미실현)**

조회 시점의 현재가를 기준가로 사용한다. 

```
위반 매수: 20개 × 1000원
매도 없음

loss_amount = (1000 - 현재가) × 20 (변동)
```
